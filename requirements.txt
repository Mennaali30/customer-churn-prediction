# NOTES:
# - pandas is capped at <3.0 because pandas 3.0 enforces Copy-on-Write,
#   under which `df[col].fillna(value, inplace=True)`-style chained
#   assignment silently no-ops instead of raising. This project's code
#   uses `df[col] = df[col].fillna(value)` instead, which is CoW-safe and
#   forward-compatible with pandas 3.0+, but the pin below is kept as an
#   extra safety net for reproducibility.
# - numpy is given a floor, not an exact pin. numpy==1.26.4 only has
#   prebuilt wheels through Python 3.12 — on Python 3.13+ pip falls back
#   to compiling from source, which fails on machines without a C compiler
#   (Visual Studio Build Tools on Windows, Xcode CLI tools on macOS).
#   Letting pip pick a compatible numpy avoids that build entirely.
#   Recommended: use Python 3.11 or 3.12 for the smoothest install.
